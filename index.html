Fama-French 5 Factor Model Data and Validity¶

Let's start by getting our Fama-French 5 Factor Model Data, cleaning it, processing it, and validating it using an existing Stock Market Returns. FF5 data is essentially derived from Total Stock Market Data and other classifiers for stock. Therefore, validating involves simply checking whether the first factor (Market Risk Premium) is equivalent to Market Return (both daily).

The Five Factors in the model are:

  1. Market Risk (MKT): This factor represents the excess return of a broad market portfolio over a risk-free rate. It captures the general risk associated with the overall stock market.
  2. Size (SMB): This factor reflects the tendency of small-cap stocks to outperform large-cap stocks. It's measured by the difference between the returns of small and large-cap stocks.
  3. Value (HML): This factor captures the outperformance of value stocks (those with a high book-to-market ratio) over growth stocks.
  4. Profitability (RMW): This factor reflects the tendency of companies with high operating profitability to outperform those with low profitability.
  5. Investment (CMA): This factor captures the effect of investment intensity on returns. It suggests that companies with low investment (conservative) tend to outperform those with high investment (aggressive).

Portfolio Construction and Statistics¶

Using the adj_close DF, we can compute our daily price returns using a simple function pct_change(). We could also fill NaNs with 0 simply because if there were no returns for the day or the stock did not yet exist, it means we could not have it in our portfolio. However, that would impact our regression and analysis, as it would essentially make that part of the portfolio stagnant until the stock goes public.

Rather, let's deal with it more realistically. Where we only include it in our equal weighted portfolio if it's value is not NaN -- i.e., the stock is public. The portfolio automatically adjusts its weights when the stock is made public and we purchase it.

In [ ]:
# We create a series (DF) with the price changes
daily_rets = adj_close.pct_change()

# Then create an empty DF of our portfolio with daily returns only on tickers in our respective portfolio and if the stock is public
port_dynamic = pd.DataFrame(index=daily_rets.index)

# For each item in the portfolio
for owner, tickers in portfolios.items():
    # Broadcasting: take the row‑wise mean, skipping NaNs automatically and constructing our portfolio only using available stock
    # This allows us to dynamically adjust the portfolio and weights to an equally weighted portfolio when a stock goes public
    port_dynamic[owner] = daily_rets[tickers].mean(axis=1, skipna=True)

# Cleaning if need be
port_dynamic.fillna(0, inplace=True)

factors = factor_list

# empirical factor stats
F = ff5_df[factors]
mu_F, cov_F = F.mean().values, F.cov().values

# simulate factor paths
sim_F = rng.multivariate_normal(mu_F, cov_F, size=(n_sims, n_days))

# draw daily RF from history (vector of shape (sims, days))
rf_hist = ff5_df['RF'].values
sim_rf  = rng.choice(rf_hist, size=(n_sims, n_days))

# α and β
alpha_vec = ff5_summary['Alpha'].values                    # (owners,)
beta_mat  = ff5_summary[[f'Beta_{f}' for f in factors]].values  # (owners, factors)


# excess → total return: r = α + β·F + RF + ε
excess = np.einsum('sdf,of->sdo', sim_F, beta_mat)          # β·F

# build idiosyncratic shocks for each owner
eps = np.zeros_like(excess)
for i, owner in enumerate(owners):
    eps[:, :, i] = rng.normal(0, resid_std[owner], size=(n_sims, n_days))

total = excess + alpha_vec[None,None,:] + sim_rf[:,:,None] + eps
# cumulative simple returns
cum = np.cumprod(1 + total, axis=1) - 1
sim_paths = {owners[i]: cum[:,:,i] for i in range(len(owners))}

# year-end summary (already 1-yr cumulative)
year_end = pd.DataFrame({o: p[:,-1] for o, p in sim_paths.items()})
